# Contributing to Autokirk MCP Wrapper

Thank you for your interest in contributing to the Autokirk ecosystem.

## Ways to Contribute

- Improve documentation
- Add or refine MCP tools
- Enhance error handling and observability
- Extend support for additional Autokirk Engines and Agents

## Development Workflow

1. Fork the repository.
2. Create a feature branch:
   ```bash
   git checkout -b feature/my-improvement
   ```
3. Install dependencies:
   ```bash
   npm install
   ```
4. Run and test locally:
   ```bash
   npm start
   ```
5. Commit your changes with clear messages.
6. Open a Pull Request describing:
   - What changed
   - Why it changed
   - Any relevant screenshots or logs

Please ensure changes are consistent with the architectural direction
of the Autokirk Operating System.
